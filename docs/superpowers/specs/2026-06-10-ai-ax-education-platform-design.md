# AI/AX 교육 플랫폼 — 설계 문서 (Design Spec)

- **작성일**: 2026-06-10
- **상태**: 설계 승인 대기 (Draft for review)
- **한 줄 요약**: 일반계 고등학생 대상 **4시간 생성형 AI 특강**을 위한 통합 웹 플랫폼. 강사 동기화 슬라이드 + 풍부한 핸즈온 실습 + Kahoot형 라이브 퀴즈를 하나로 묶고, 로그인 없이 강의실 토큰+닉네임으로 입장한다.

---

## 1. 목표와 배경

- **교육 대상**: 일반계 고등학생 (비개발자)
- **분량**: 4시간 (240분)
- **핵심 메시지**: "주문(프롬프트)을 외우지 말고 **맥락을 줘라** — 그리고 AI는 너의 진로에 *뼛속까지* 들어온다."
- **운영 환경**: 단체 교육. 로그인 없음. 강의실 토큰으로 입장. **같은 IP(교실 NAT)에서 다수 접속**. 학생은 **스마트폰·태블릿 비중이 큼**.
- **사용 가능한 AI**: MiniMax API(텍스트/대화 등), Stability API(이미지 생성).

### 설계 우선순위
1. 교육 콘텐츠(커리큘럼) 우선 확정 → 그에 맞는 슬라이드/실습 구성.
2. 미성년자 대상이므로 **콘텐츠 안전장치**와 **비용/쿼터 제어**를 1급 요구사항으로 둔다.
3. 모바일 우선 반응형, 한국어 UI.

## 2. 교육 커리큘럼 (4시간 / 240분)

| # | 시간 | 파트 | 핵심 내용 | 핸즈온 / 인터랙션 |
|---|---|---|---|---|
| 0 | 15분 | 입장 & 워밍업 | 토큰→닉네임, 아이스브레이킹 | 단어 투표 + Kahoot 워밍업(OX) |
| 1 | 35분 | 생성형 AI의 정체와 특성 | 예측 원리, 종류(글·그림·음성·영상), 잘하는 것 vs 못하는 것, 환각 | AI 첫 대화 + Kahoot 퀴즈 |

| 7 | 15분 | 마무리 & 종합 대회 | 전체 복습 | 종합 Kahoot 토너먼트 + 소감 투표 + 시상 |

합계 240분. 파트 4·6은 진도에 따라 ±5분 버퍼.

### 후반 메시지 포인트 (슬라이드 콘텐츠에서 구체화)
- **프롬프트가 덜 중요해진 이유**: 프런티어 모델은 표현 차이에 강건해졌다. *어떻게* 말하느냐보다 *무엇을(어떤 맥락을)* 주느냐가 결과를 가른다.
- **AI학과가 하는 일**: 모델 개발자만이 아니라 데이터, 서비스 기획, AI 윤리/정책, 의료·법률·예술 융합 등 스펙트럼이 넓다.
- **진로에 뼛속까지**: 직업이 사라지기보다 "AI를 쓰는 사람"으로 재편된다.
- **비전공자도 잘해야 하는 이유**: 코딩이 아니라 **맥락을 잘 주는 능력**(=이 강의의 내용)이 경쟁력. 전공 무관.

### 비유 (고등학생 눈높이)
- 컨텍스트 = "심부름 전에 배경 정보·예시를 쥐여주기"
- 하네스 = "한 번에 다 시키기 vs 단계로 쪼개 시키고 도구까지 들려주기"

## 3. 시스템 아키텍처

### 스택
- **프론트엔드**: React + Vite SPA, 모바일 우선 반응형, 한국어 UI, Socket.IO 클라이언트
- **백엔드**: Node + Fastify — 빌드된 SPA 서빙 + REST API + Socket.IO + AI 프록시. Oracle Cloud VM 단일 프로세스(앞단 nginx).
- **DB**: Supabase (Postgres) — 시스템 오브 레코드. (스키마 생성/마이그레이션은 Supabase MCP로 수행)
- **AI 호출**: MiniMax/Stability 키는 **서버 환경변수에만** 보관. 모든 호출은 서버 프록시 경유(브라우저 키 노출 절대 금지).

### 화면(역할) 3종
- 🧑‍🏫 **강사 콘솔**: 강의실 생성, 슬라이드 넘김, 실습/퀴즈/랩 열고 닫기, 모니터링, 리더보드, 패닉 컨트롤
- 🧑‍🎓 **학생 뷰**: 토큰+닉네임 입장 → 동기화 슬라이드 + 핸즈온 + 퀴즈 응답 (모바일 우선)
- 📺 **프로젝터 뷰**(선택): 워드클라우드·리더보드 등 큰 화면 표시용

### 권한 모델 (로그인 없음)
- **강사**: 강의실 생성 시 서버가 `instructor_secret` 발급 → 강사 브라우저에 저장. 이 시크릿 보유자만 강의실 조종.
- **학생**: 강의실 토큰 + 닉네임 → 서버가 `session_id` 발급(브라우저 저장). 같은 IP라도 개인 식별·집계·쿼터 가능.

### Supabase 스키마 (초안)

| 테이블 | 핵심 컬럼 | 용도 |
|---|---|---|
| `classrooms` | id, token, deck_id, status, current_slide, current_activity, instructor_secret, created_at, expires_at | 강의실/진행 상태 |
| `participants` | id, classroom_id, nickname, session_id, score, joined_at | 학생·점수 |
| `quiz_responses` | id, classroom_id, participant_id, question_id, answer, is_correct, response_ms, points | Kahoot형 채점/속도 점수 |
| `poll_responses` | id, classroom_id, activity_id, participant_id, value | 투표/워드클라우드 |
| `ai_usage` | id, classroom_id, participant_id, type(chat/image), units, est_cost, created_at | 1인당 쿼터·비용 추적 |
| `lab_runs` | id, classroom_id, participant_id, lab_type, input, config, output, created_at | 비교 랩 결과 로깅(선택) |

- **강의 콘텐츠(슬라이드·퀴즈 문제)** 는 리포지토리 내 **JSON/TS 덱 파일**로 정의(`deck_id`로 참조). 버전관리 용이, 재시작에 안전. DB에는 콘텐츠를 넣지 않음.

### AI 프록시 엔드포인트 (초안)
- `POST /api/chat` — MiniMax 텍스트/대화
- `POST /api/image` — Stability/MiniMax 이미지
- `POST /api/lab` — A/B 비교 랩(같은 입력, 두 설정 동시 실행)
- 모든 호출 전 **① 쿼터 확인 ② 콘텐츠 안전필터** 통과 필수.

## 4. 실시간 동기화 모델 (Socket.IO)

- **방(room)** = 강의실 1개. 서버가 `current_slide`, `current_activity`의 권위자.
- **강사 → 서버**: `slide:goto`, `activity:open/close`, `quiz:start/next/reveal`
- **서버 → 학생**: `slide:changed`, `activity:opened`, `quiz:question`, `quiz:reveal`, `leaderboard:update`, `participants:count`
- **학생 → 서버**: `join`(토큰+닉네임), `quiz:answer`, `poll:vote` *(AI 호출은 REST 프록시로 분리)*
- **재접속 복구**: 끊겼다 재접속하면 서버가 현재 상태 스냅샷(현 슬라이드+열린 활동)을 즉시 전송 → 늦게 들어온 학생도 자동 동기화.
- 진행 상태는 Supabase에 미러링하여 서버 재시작/늦은 입장에 대응.

## 5. 실습 랩 프레임워크

> 상세 카탈로그(실습 종류 다수)는 구현 단계에서 확장한다. 여기서는 골격만 정의.

**활동(activity) 레지스트리** 구조 — 새 실습을 플러그인처럼 계속 추가 가능.

초기 유형:
- 💬 **AI 대화**(MiniMax): 자유 + 미션형(가이드 프롬프트)
- 🎨 **이미지 생성**(Stability/MiniMax): 키워드·스타일 변주
- 🔄 **반전 실험**: 표현만 바꾼 프롬프트 결과 비교
- 🆚 **A/B 비교 랩**: 같은 입력 → 두 설정(컨텍스트/하네스) → 결과 나란히 (재사용 1종으로 파트 2·3·4 커버)
- 🎮 **Kahoot형 퀴즈**: 정답 + 속도 점수 + 리더보드 + 랭킹 + 최종 우승자
- 🗳️ **투표/워드클라우드**: 의견 수렴(정답 없음)

라이브 인터랙션은 위 3가지 모드(투표 / Kahoot 퀴즈 / A/B 랩)로 시스템화하여 재사용한다.

## 6. 안전 · 비용 제어 (미성년자 필수)

- **콘텐츠 안전필터**: 서버측 프롬프트 검열(금칙어 + 제공자 안전옵션) → 차단 시 친절한 안내 메시지. 이미지 생성은 안전 스타일·필터 적용.
- **1인당 쿼터**: 활동별 채팅 N회 / 이미지 N장 (강사 설정 가능) — `ai_usage`로 집계·차단.
- **강의실 전체 예산 상한**: 누적 비용 한도 초과 시 자동 차단.
- **강사 패닉 컨트롤**: 전체 AI 일시정지 · 강의실 잠금 · 초기화.
- **프라이버시**: 닉네임 외 개인정보 미수집. `expires_at` 경과 후 강의실 데이터 자동 정리.
- **레이트 리밋**: 서버 단에서 IP/세션 기준 요청 제한(같은 IP 다수 접속 고려해 세션 우선).

## 7. UI / 반응형

- **모바일 우선**: 스마트폰·태블릿 다수. 큰 터치 타깃.
- **한국어 UI**.
- **Kahoot 보기**: 큼직한 색/도형 버튼.
- **슬라이드**: 화면 크기에 맞춰 스케일(프로젝터/데스크톱/모바일).
- 상세 화면 시안(목업)은 구현 단계에서 작성.

## 8. 범위 밖 (Out of Scope, 이번 버전)
- 회원가입/로그인/지속 계정
- 강사용 콘텐츠 위지윅 편집기 (콘텐츠는 코드 내 덱 파일로 관리)
- 결제/과금 시스템 (API 비용은 운영자 키로 충당, 쿼터로 통제)

## 9. 미해결/추후 결정
- 실습 랩 상세 카탈로그(실습 종류 다수) — 구현 단계에서 확장 (사용자 요청: "실습 기능 많아야 함")
- MiniMax/Stability 정확한 모델·파라미터·비용 단가 → 구현 시 API 문서 확인
- 강의실 동시 인원 상한, 쿼터 기본값(N) 구체 수치
- 프로젝터 뷰 포함 여부 최종 확정
